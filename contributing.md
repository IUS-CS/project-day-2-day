## Code Review Standards

Below are the code review standards:

* Code should contain as minimal errors/bugs
* Clear descroptions on what was changed, modified, or added
* At least 75% of members in agreeance
* Must have unit tests OR a method of recreating.

## Tools to employ

* Python is a necessary must.
* Either IntelliJ or a IDE of their choice.
* GitHub is a must
* Flask and FastAPI is perferred
* PyLint 

## AI Policy

Whilst we make all attempts for AI Generative content to not be a part of the project, as a group we understand that the use of the technology as a tool is much more 
beneficial if used as a resource and not a copy and paste machine. Under circumstances of debugging and understanding what the code that we are writing is doing and/or
how the code works with other parts of the src code; we may use AI to better explain and help with general understanding. We may also use AI to help with the generation
of ideas or thoughts about where the project is going and whether or not that is in the interests of the product as a whole. Again, AI is a tool and we will treat it as
such.
